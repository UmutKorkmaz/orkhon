# Old Turkic (Göktürk) data

The realistic Göktürk capability for Orkhon is a **transliteration / translation
assistant**, not a fluent speaker — the entire attested corpus (Orkhon + Yenisei +
Tonyukuk inscriptions) is only ~a few hundred lines. See `docs/turkic-languages.md`.

## What's here (deterministic, correct-by-construction)

- `translit_sft.jsonl` — synthetic **rune → Latin transliteration** SFT examples,
  generated by `orkhon.data.old_turkic.synth.make_translit_sft`. The rune↔Latin map
  is authoritative (built from the Unicode Old Turkic block U+10C00–U+10C4F), so
  every example is exactly correct. This teaches the *script*, no scholarship needed.

## What is intentionally NOT here (needs sourcing — never invented)

- **Translation** pairs (Old Turkic → modern Turkish / English). These require real
  scholarly attestations and licensing:
  - the [Uppsala runiform database](https://www.runiform.lingfil.uu.se/) (CC BY-NC-SA
    — commercial weights need review),
  - critical editions (Talât Tekin, Marcel Erdal, Clauson's lexicon).
  Add them to an `inscriptions.jsonl` with fields: `inscription`, `line_id`,
  `old_turkic_unicode`, `transliteration`, `modern_turkish`, `english`, `notes`,
  `license`, `confidence`. The model is never trained on invented meanings.

## Next step

After the `bengü` Turkish base finishes training, SFT it on `translit_sft.jsonl`
(the transliteration task) → a model that reads Old Turkic runes. Translation is a
later step, gated on sourced data.
